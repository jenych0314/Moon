# 3. 날짜형
<table>
    <thead>
        <tr>
            <th>데이터 타입</th>
            <th>바이트 수</th>
            <th>표현 범위</th>
            <th>설명</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>DATE</td>
            <td>3</td>
            <td>1000-01-01 ~ 9999-12-31</td>
            <td>날짜 표현</td>
        </tr>
        <tr>
            <td>DATETIME</td>
            <td>8</td>
            <td>1000-01-01 00:00:00 ~ 9999-12-31 23:59:59</td>
            <td>날짜와 시간 표현</td>
        </tr>
        <tr>
            <td>TIMESTAMP</td>
            <td>4</td>
            <td>1970-01-01 00:00:00 ~ 2037-01-19 03:14:07</td>
            <td>INSERT, UPDATE 연산에 유리</td>
        </tr>
        <tr>
            <td>TIME</td>
            <td>3</td>
            <td>-838:59:59 ~ 838:59:59</td>
            <td>시간 표현</td>
        </tr>
        <tr>
            <td>YEAR</td>
            <td>1</td>
            <td>1901 ~ 2155, 70 ~ 69 (1970~2069)</td>
            <td>연도 표현</td>
        </tr>
    </tbody>
</table>
YEAR(4) 와 같이 명시적인 길이를 표기한 데이터 유형은 MySQL 8.0.19 이후 버전부터 사용되지 않습니다.
YEAR(2) 와 같이 두 자리로 표기하는 데이터 유형은 MySQL 5.7 이후 버전부터 지원하지 않습니다.
